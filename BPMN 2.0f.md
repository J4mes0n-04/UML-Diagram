# Предприятие заполняет онлайн-форму на сайте с описанием своей технологической проблемы.
## 1. Валидация заявки.
-Администратор проверяет полноту данных (если неполно → запрашивает доп. информацию у предприятия)
-Проверяет соответствие тематике (если не соответствует → отклоняет с объяснением)

## 2. Дополнительные условия:
-Процесс должен быть завершен за 7 рабочих дней
-При превышении срока отправляется эскалация руководителю
-Эксперт может запросить дополнительную информацию у предприятия на этапе экспертизы
-Все шаги должны логироваться

## 3. Классификация
-Валидную заявку классифицируют по отрасли (машиностроение, металлургия, ТЭК).


## 4. Экспертиза
-Эксперт изучает проблему и ищет подходящие решения в базе.

## 5. Распределение экспертам
-Заявку автоматически назначают свободному эксперту нужного профиля
-Если свободных нет → заявка попадает в очередь ожидания

## 6. Результат
-Если решение найдено то формируется предложение с вариантами
-Если решения нет то заявка передается в R&D-отдел для исследования

## 6. Финализация
Предприятию отправляется финальный ответ (предложение или уведомление об исследовании).

A1. Получена новая заявка от предприятия
A2. Запрос информации у предприятия
А3. Заявка обработана
